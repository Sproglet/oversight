Oversight (nmt jukebox) & unpak.sh post processor for nzbget

 

1 May 2009: Note the current beta has some fixes

( Get the cross platform nzbget post processing scripts only )

This is another jukebox but I think it is significantly different from other current offerings and it was born out of necessity for me.

Main Goals and features:

  • quicky find whatever has recently downloaded with no searching or browsing at all.
  • Give me a an overview of what content I have without too many page draws (NMT is slow for non-video processing)
  • Hands free, zero-touch indexing, - no manual renaming of files to help the application do its work
  • No dependency on a PC for indexing or generation of html etc.
  • Load and go. Default configuration will index internal HD
  • Looks good on SD. I have a 36 CRT, so the main overview is mostly text to save space. Some basic artwork is displayed on drill down. This is unlikely to change. Other video juke boxes are pretty, but not on SD. The NMT is not a fast platform for scrolling many pages of large DVD artwork on an SD screen. I have added some HighDef detection for a better layout in Flat Panel tvs.
  • make as few assumptions as possible about the physical layout of folders, no moving files around to help the application work.
  • Although there is now a full file renaming capability in unpak.sh/catalog.sh, it is not really needed anymore if using OverSight, I would only use it to change folder names. File names provide useful information if you have to ever rebuild from scratch.

Usage notes:

  • You need an internet connection to your PCH
  • You should not have to change anything if indexing your internal HD - famous last words. Indexing other drives doesnt work at present. Probably will work with a single line edit.
  • There will be some mis-identified files. Main culprits are
    • ISOs of TV box sets (I havent added any specific detection for that)
    • Programs showning in multiple countries , The Office, Kath and Kim
    • Any file with an odd naming convention
    The best solution is to provide a manual fix up but low priority for now.
  • Colour buttons change sort order and TV/Film filtering
  • [Blue] stuff is TV, [Green] is Film , [Red] is - not sure
  • The [Remove] option just removes files from the index - they are not deleted
  • The [Delete] option will delete files.
  • There is a text message filter that works by selecting the number keys. When you see it it should make sense! There is room for improvement.

The application has three parts:

  • unpak.sh - Only does "intelligent" unpacking.
  • catalog.sh - Indexes video files and also has rename capability. Used by unpak.sh
  • oversight 'WebService' - This is specific to the NMT platform and provides a web gui for the index built by catalog.sh

The index is built by searching English Language sites imdb,google and epguides. If you would prefer to use the folder titles in the index run the following commands : (once only) 
su -s /bin/sh nmt
cd /share/Apps/oversight
echo "FOLDERTITLES=1" >> catalog.cfg

#Rebuild index
rm index.db index.db.idx
./catalog.sh FORCE IGNORENFO /share/
